# High-Level Architecture (HLA)

This document describes the high-level architecture and interaction of abstractions in the project.

`spectask-init` parses `--ide` as one or more IDE keys (`argparse` `nargs="+"`), except that `--ide` may be omitted when `--update` is passed (then the CLI behaves as if `--ide auto` were given). The flag `--update` forces `--skip-example` and `--skip-hla-file` only; it does not imply `--skip-navigation-file`. A standalone `--skip-hla-file` omits copying `spec/design/hla.md` from the template’s required list (same pattern as `--skip-navigation-file` for `bootstrap.NAVIGATION_FILE_RELPATH`, i.e. `spec/navigation.yaml`). Before copying required paths, `run_template_bootstrap` checks `required-list.json`: if it would copy `spec/navigation.yaml` or `spec/design/hla.md` and that path already exists in the working directory (and the corresponding skip flag is not set), it raises a `RuntimeError` with guidance to use `--skip-navigation-file` and/or `--skip-hla-file` as needed, or `--update` when skipping example and HLA copies is enough (or `--update --skip-navigation-file` when both that shorthand and skipping the registry are intended). `bootstrap.ide_files_for` merges the path lists from `skills-map.json` in key order without duplicate paths. The keys `all` and `auto` are exclusive (each must be the only value after `--ide`). For `auto`, `bootstrap.resolve_auto_ide_keys` reads the acquired template’s `.metadata/ide-detection.json`, matches `file`/`directory` markers under the current working directory (OR within each IDE entry), and yields every matching IDE name in detection-file order; `ide_files_for` then merges those lists like explicit multi-key `--ide`.

After required/example/IDE copies inside `run_template_bootstrap`, `bootstrap.apply_template_migration` runs if the acquired template contains `.metadata/migration.json`. The file may declare an ordered `operations` list (`type`: `move` with `from`/`to`, or `delete` with `path`) or legacy top-level `move`/`delete` arrays (all moves, then all deletes). Destructive steps quarantine existing paths under `.backup_spectask/{name}_{uuid}` and emit English warnings plus a one-time `.gitignore` hint on stderr. Immediately afterward, `bootstrap.reconcile_navigation_with_spec_tree` updates `spec/navigation.yaml` if present: its `extend:` and `design:` lists are aligned with `spec/extend/**/*.md` and `spec/design/*.md` (auto-append missing paths with placeholder descriptions, drop entries for missing files, English stderr warnings). Navigation I/O uses PyYAML (`pyproject.toml` runtime dependency).

When `--extend` is used, `bootstrap.run_extend` runs `merge_extend_source_navigation` (merges only the extend bundle’s `extend:` list into the cwd registry when both sides have `spec/navigation.yaml`; non-empty `design:` in the bundle is warned and not merged), then `copy_extend_overlay` (still only `spec/extend/**` from the bundle), then `reconcile_navigation_with_spec_tree` again. `run_template_bootstrap` and `run_extend` accept an optional `stderr` sink; the CLI passes `sys.stderr`.

**Tests:** `pytest` integration cases (marked, network for the canonical GitHub template ZIP) call `cli.main()` with `--ide auto` and with `--update` and no `--ide`, in addition to existing ZIP/Git template and skip-flag coverage.

**Publishing:** `scripts/publish.py` bumps the patch version in `pyproject.toml`, removes existing `*.whl` and `*.tar.gz` under `dist/` (leaving e.g. `dist/.gitignore`), then runs `uv build` and `uv publish` so PyPI uploads only the new artifacts.
